決策樹 (Decision Trees) 被廣泛用於分類與迴歸資料。藉由一系列的 if/else 問題，對資訊進行分類。雖然如此，輸入並非一題一題而是一次給予所有問題的答案，而是經由一系列的 if/else 問題所產生的答案 (不管是人工或是透過程式)；類似下圖，藍色圓形與紅色三角形個代表一筆資料；為了表示每一個點的座標，$X[0]$ 表示橫坐標而 $X[1]$ 表示縱座標：

# 分類

接著演算法將試著將訓練資料分成兩部分，即 postive 與 negtive 兩塊；然後將兩區塊再分為兩個子區塊。每區隔一次，決策樹就增加一層 (depth += 1)。決策樹的 root 即為 depth 0，為所有資訊的集合；第一層為 depth 1，其 if/else 問題為 $X[1] <= 0.0596$。

第二層因為有兩個區塊，各區塊有各自的 if/else 問題 ($X[0] <= 0.4177$ 與 $X[0] <= 1.1957$)，各自將區塊再分為兩個子區塊。我們可以看到，子區塊中的 postive 區塊集合，即為類別 Ａ (藍色圓形)；而 negative 區塊的集合，即為類別 B (紅色三角形)。 以此類推，當決策樹越深，分類將會會精準。決策樹將會持續相同的做法，直到兩個子區塊間只包含同一類別；只包含單一類別的區塊被稱為 pure。
![[Screenshot 2026-03-16 at 10.58.32 AM.png]]

當決策樹的決策邊界建立之後，新資料只要比對所在位置的子區塊劃分為哪一個類別即是分類的方法。
# 迴歸

流程等同分類，當找到所在子區塊後，將該子區塊中==所有類別==的值平均之後，即為新資料的迴歸預測。

# 結論

決策樹最終的結果將會是非常複雜的模型，因此容易成為過似合。為了避免這種情況，有兩種方法：

1. 預剪枝 (Pre-Pruning) 
    - 藉由限制決策樹的深度、子區塊的數量、或是每個區塊中雜質的最少數量，提前結束演算法
2. 後剪枝 (Post-Pruning)
    - 將決策樹最終的結果，移除極少量資訊的節點 (與 if/else 問題) 

> [!NOTE]
> 即便有了預剪枝 (Pre-Pruning)，決策樹的過似合比例還是高於其他演算法。解決法案請參考[決策樹集成](Ensembles%20of%20Decision%20Trees.md)一節

## 特徵重要度 (feature importance)

決策樹演算法最重要的就是盡可能減少決策樹的深度，而減少深度的方式就是把最關鍵的 if/else 問題放在前面。當驗證演算法時，應該計算每個 if/else 被呼叫的次數；累計的次數愈多，表示該 if/else 問題越關鍵。

當調整 if/else 問題時，資料的輸入順序亦要跟著調整。有時候藉由調整資料，包含移除無意義的資料或是改變資料的順序，可以簡化決策樹。

# 參考資料

決策樹在機械學習中是非常強大的演算法，廣泛的被使用。他不像[線性模型](Linear%20Models.md)需要調整每個參數的係數就可以正常運作，也不需要將各參數的值轉換成同一計量單位 (scaling of the data)；他唯一的缺點就是過似合的比率過高，而這個問題將在[決策樹集成](Ensembles%20of%20Decision%20Trees.md)中被解決。

目前在 [scikit-learn](https://scikit-learn.org/stable/) (使用 Python 的機械學習架構) 提供兩種演算法，該兩種演算法僅支援預剪枝：
1. 實作於 sklearn.tree 中的 DecisionTreeClassifier 類別
2. 實作於 sklearn.tree 中的 DecisionTreeRegressor 類別
