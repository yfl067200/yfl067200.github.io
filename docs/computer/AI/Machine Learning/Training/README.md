
# 準備工作

機械學習最大的困難在於，找出解決問題的演算法與合適的輸入與輸出 (Input and Output)。你給予的資訊必須是譨讓演算法理解，且能合理推算出正確結果的樣本組合。之後將會有章節說明如何建立好樣本的方法，為[特徵工程](docs/computer/AI/Feature%20Engineering/README.md)中的重點

> [!Note]
> 在機器學習中，表格中的行被稱為樣本 (sample 或是 data point)，列被稱為特徵 (feature)。

不論如何，你都必須在訓練之前理解你所使用的演算法，並檢視你使用的樣本資料。以下幾點可以作為開始訓練前的幾項檢視條件：

- [ ] 你要如何解決目前的問題？
  - [ ] 如何透過機械學習的方式，說明你的問題
  -  [ ] 我的樣本可以被演算法理解，且導出正確結論？
  - [ ] 我收集的樣本數量，足夠讓機械學習進行正確的預測？
   -  [ ] 如何判斷機械學習完成訓練
   -  [ ] 哪一項特徵是我希望機器學習能夠從資料中找到，並被演算法使用
- [ ] 如何將訓練好的機器學習方案互動？

# 訓練方式

目前最成功的訓練方式為[監督式學習](./Supervised%20Learning) (supervised learning)。藉由提供 Input 與 Output，讓演算法自行找出從 Input 得到"正確" Output 的方法。

問題在於，Input 與 Output 組合與演算法所要解決的問題相關，因此提供正確的 Input 與 Output 組合將影響至鉅。舉一個例子，提供相同的男女的照片 (Input)，要找出辨識性別或是辨識是否站立的 Output 是不同的。

相較於監督式學習，另一種訓練方式為[非監督學習](./Unsupervised%20Learning) (unsupervised learning)。在缺乏明確 Output 的情況下，要讓演算法從 Input 中學習並獲得正確的 Oupout 將會是複雜且難以控制。

不論是哪一種訓練方式，都必須確保 Input 將會是可以被理解的。通常在決定 Input 前，會先建立一個表格；將每一行代表一個 Input，再將該 Input 對演算法有影響的資訊作為列。有了這個表格之後，使用者即可針對每一次學習得出的結果與預期的結果進行檢驗。

# 訓練結果

透過訓練，機械學習可能會產生三種結果：

| 訓練結果                                      | 說明                                              |
| :---------------------------------------- | ----------------------------------------------- |
| 正規化 (Generalization)                      | 不論是使用訓練樣本或是對驗證資料進行預測，成功機率的差異最少                  |
| 過似合 (Overfitting)<p>過訓練 (Overtraining)    | 使用訓練樣本的成功機率非常高，但是對驗證資料進行預測的結果不佳<br>常見於==複雜==的模型 |
| 欠似合 (Underfitting) <p>欠訓練 (Undertraining) | 不論是使用訓練樣本或是對驗證資料進行預測，成功機率都非最佳<br>常見於==簡單==的模型   |
因為機械學習的骨幹就是統計，所謂的訓練就是透過訓練資料建立一個常態分佈曲線，或稱為==決策邊界  (Decision Boundary)==。當該決策邊界越複雜 (越不平滑)，我們稱該模型越==複雜==；相反的，越是平滑的決策邊界，代表該模型越==簡單==。越複雜的模型，表示該決策邊界與訓練資料越貼近，容易成為過似合的狀況。過於簡單的模型又可能忽視訓練資料的特徵，進而成為欠似合。

通常為了確保我們的結果符合正規化，我們樣本會準備不只一組；部分用來進行訓練，其他將用來驗證。為了得到最佳化的結果，會調整參數甚至會改用訓資料進行多次訓練。之後會用訓練後建立的模型，進行驗證的動作。最後將兩者的成功率進行比對，差距最少的將成為最佳解。
