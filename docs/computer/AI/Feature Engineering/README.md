# 資訊類別


## 合理性檢查 (Sanity Check)

因為方便，進行機械學習時會將特徵數值化；數值化資訊的合理性檢驗 (sanity check) 所需要考慮的點將包含以下幾點，除了過濾掉不必要的雜質，也盡可能提高演算法的效能：

- 有沒有正負號會造成影響嗎？
- 使用的刻度為何？
    - 需要考慮細微的差異嗎？
	    - 當數值差距極大時，刻度可能改為 ${10^N}$ 可能比較恰當
    - 或是使用級數表示一個群組的值也可以？
	    - 譬如說年齡可以將每 10 年為一單位，將人分為多個群組
- 改成次數而非一般的數值，是否更能表現出特徵？

而非數值話的資訊所需要考慮的點不盡相同，但是基本的核心都是：

1. 除去雜質
2. 提高演算法效能
