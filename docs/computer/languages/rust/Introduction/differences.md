雖然 rust 源於 c/c++，但是 rust 的設計與語法跟 c/c++ 有顯著的差異。

# 變數的可變性

在 c/c++ 宣告的變數，預設可以修改其值；而不可修改值的變數，在宣告時需要加上關鍵字 `const`。但是 rust 與其相反，變數的值預設是不可修改的，我們稱這些變數為 `immutable variable`；需要調整變數的值，在宣告變數時，需要加上關鍵字 `mut`。

``` rust
# c/c++ 宣告變數
int i = 3;           #    預設變數 i 的值可以改變

const int j = 5;     #    預設變數 j 的值不可以改變

# rust 宣告變數
let i: i32 = 3;      #    預設變數 i 的值不可以改變

let mut j: i32 = 5;  #    預設變數 j 的值可以改變
```

如果你試圖調整 immutable 變數的值，rust compiler 會返回下列錯誤警告

> cannot assign twice to immutable variable

> [!NOTE]
> 另一個關於重新賦予變數值需要注意的是，rust 畢竟是源自於 c/c++ 而非 python，所以重新指派變數值需要注意變數的型態。指派的值如果無法自動轉型成符合變數的型態，則指派的動作會失敗。

## Const

rust 亦有 `const` 關鍵字，建立的亦是不可修改值的變數。與一般的變數不一樣的地方包含：
 - 一般變數宣告時使用 `let` 關鍵字，而 const 變數使用 `const` 關鍵字
 - 一般變數可以使用 `mut` 關鍵字，成為 mutable 變數；但是 const 變數不可搭配 `mut`
 - 一般變數宣告時可以不加上型態註釋 (annotated)，如 `let i = 3`；但是 const 變數需要加上型態註釋，如 `const i: i32 = 3`
 - const 變數的值必須在編譯期間即可得出，不可包含執行期才會得到值。

因為 const 變數如此特殊，因此 rust 建議該變數的名稱使用==全大寫==，不同字之間使用==底線 (\_) ==連結。因為 const 變數在編譯時就會得知其值，所以 const 變數與值會變成執行檔的一部分，存放在與一般變數不同的區塊。

## Shadowing

在==宣告==變數時，如果變數的名稱與另一個已存在的變數相同，將導致已存在變數被遮蔽 (shadow) 的狀況。程式碼將使用新宣告的變數，而非原本已存在的變數。遮蔽的效果將結束於另一個遮蔽，或是該程式碼區塊結束。

> [!Note]
> 因為這個流程很像是重新指派變數的值，導致該變數被誤認為 `mutable 變數`，需要特別注意。

``` rust
fn main() {
    let x = 5;            //    第一個同名變數

    //    這邊是宣告變數，而非重新指派變數的值。特別容易將變數視為 mutable 變數
    let x = x + 1;        //    第二個同名變數，遮蔽第一個同名變數
    {
        let x = x * 2;    //    第三個同名變數，遮蔽前兩個同名變數
        println!("The value of x in the inner scope is: {x}");// 使用第三個同名變數
    }

    println!("The value of x is: {x}");  // 使用第二個同名變數
}
```
